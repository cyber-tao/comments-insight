# 项目说明
这是一款Chrome扩展，用来分析用户的评论，支持Youtube, Bilibili, Weibo, Douyin, Twitter, Tiktok, Reddit。

# 功能需求
- 可以提取当前页面中，帖子主题和用户所有的评论
- 评论要包含用户名、评论时间、点赞数，还有它的所有回复（同样包含和评论一样的信息）
- 可以使用设定好的AI的API进行所有评论的分析
- 要能分析出用户的这些评论所表达的情感，还有热点评论等，分析为什么会是热点，整体要有专业的分析方法
- 评论可以导出为.csv文件，分析结果可以导出为markdown文件
- 有设置页面，可以设置获取的最大评论数，还可以设置AI的API URL，API KEY，然后可以获取模型列表，再从里面选择模型，最后可以设置模型的相关的参数，比如最大token数, 温度，top_p等
- 设置可以导出和导入
- 有历史记录，可以查看之前帖子的所有评论和分析结果，并且可以有链接打开原帖子
- 要有好的交互设计与视觉效果
- 所有的任务都可以后台进行，有任务列表管理和状态提示等，能看到用了多少时间和消耗了多少tokens，当完成的时候弹出提示
- 要支持多语言，现在先要有中文和英文版本，其它语言以后再添加，默认语言根据浏览器的语言来选择，如果没有匹配的，就默认为英语，当然用户也可以在设置里设置

# 技术要求
- 评论的获取使用AI分析的方式，把网页的内容给AI，让AI来找出评论与相关的数据在哪里并提取出来
- 网页内容给AI分析提取评论时要有好的提示词，让AI严格以JSON的格式返回结果
- 在设置里可以分开设置，哪个模型用来提取评论，哪个模型用来分析评论
- 提取评论的过程，可以先从网页外层结构，然后向内，一层层告诉AI有哪些内容，让AI决定挑选要获取哪些网页内容的结构，这样可以提高AI提取评论的效率
- 因为有些网页的评论是要滚动加载的，所以用来提取评论的AI要能控制网页的滚动，还有很多评论的回复也是折叠的，需要点击打开，同样的这个AI要能控制点击展开的操作
- 要能区分发布模式与开发模式，在开发模式下，要有详细的日志输出，方便调试
- 要生成Github自动化的流程，当有版本tag的时候，自动生成扩展版本并发布到Google的扩展商店里去
- 评论数据量可能会超过模型的最大token数，所以要有一个分析的流程，把评论分成多个部分，然后分别分析，最后再合并结果，这样就不会超过最大token数了

# 其它要求
- 总是用中文回答
- 如果要运行命令，请一行一条地运行，不要把多个命令连接成一条运行
- 生成的注释与日志输出要符合项目规范，并且都为英文
- 如果要查询开发文档与资料，可以使用context7这个MCP进行
